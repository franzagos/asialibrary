## What This App Does
A private, invite-only book library platform where each user manages their own personal collection — books are added by photographing the cover, AI identifies and enriches the record automatically.

## Users & Roles
- **Asia (admin)**: identified by `ADMIN_EMAIL` env var. Only user who can generate invite links. Has access to `/admin/invites`.
- **Invited users**: can add, edit, and delete books in their own library only. Cannot see other users' libraries. Cannot invite others.

## Authentication
- Email/password only (no Google OAuth)
- Invite-only: open registration is blocked. Users can only register via a unique invite link generated by Asia.
- First login: user lands on `/library` with an empty state CTA.

## Key Screens

### Pre-login
- Layout: centered single-column, max-w-sm
- Primary element: email + password login form
- Unauthenticated state: redirect to `/login` — no flash of content

### `/register?token=xxx`
- Layout: centered single-column, max-w-sm
- Primary element: name + email + password form; token validated server-side on submit
- Error state: "Link non valido o scaduto" if token is missing, used, or expired

### `/library` (main screen)
- Layout: fixed sidebar 240px left + scrollable main area, max-w-7xl
- Sidebar: logo/title, category list with book counts, "All books" option, divider, "Admin" link (admin only), user name + sign out
- Header: "La mia libreria" title + search bar + grid/table toggle + "+ Aggiungi libro" button (terracotta)
- Body: BookGrid or BookTable based on toggle; filter chips for active category/tag
- Empty state: centered book icon (lucide) + "Nessun libro ancora" + "Aggiungi il tuo primo libro" CTA
- Mobile: sidebar becomes a drawer (Sheet), bottom bar with key actions

### `/library/upload`
- Layout: centered, max-w-2xl, 3-step flow
- Step 1: drag-and-drop image zone + "Identifica libro" button
- Step 2: skeleton loader + "Sto identificando il libro..."
- Step 3: full BookForm pre-filled with AI results; all fields editable; "Salva nella libreria" CTA
- Error state: warning banner if AI fails, still show empty form for manual entry

### `/library/[id]`
- Layout: two-column desktop (cover left 1/3, info right 2/3), single column mobile
- Shows: cover image, title, author, year, edition, category badge, tags, description, market price, purchase status + location, personal notes, date added
- Actions: "Modifica" button (inline edit mode), "Elimina" button (confirmation Dialog)

### `/admin/invites`
- Layout: full-width table
- Header: "Gestione inviti" + "Genera nuovo invito" button (copies link to clipboard)
- Table: columns for link (copy icon), email, created date, expires date, status (Attivo/Usato/Scaduto)
- Only accessible if `session.user.email === process.env.ADMIN_EMAIL`

## Core Features
1. Invite-only registration — token-based, 7-day expiry, single use
2. AI book recognition from photo — vision model identifies title/author/year/edition, Perplexity enriches with description + market price
3. Book library with categories and tags — sidebar navigation, category filter, free-form tags
4. Dual view: cover grid + filterable table — toggle persisted in localStorage
5. Book detail with full edit and delete
6. CSV export of entire personal library

## Design System

**Layout**: fixed sidebar 240px · scrollable main · max-w-7xl
**Density**: balanced
**Accent colors**: terracotta `#C4614A` (primary CTA) · ochre `#D4A853` · sage `#7A9E7E` · coral `#E8896A`
**Background**: white `#FFFFFF` · light gray `#F5F4F2` · dark text `#1A1A1A`
**Mode**: light only
**Border radius**: soft (rounded-xl for cards, rounded-lg for buttons/inputs)
**Reference apps**: Circle.so (layout + spacing), Matisse palette (color inspiration)
**Tone**: calm, editorial, personal — no playful microcopy

### Component Conventions
- **Forms**: inline on the detail page (edit mode toggle); full page for upload flow
- **Confirmation dialogs**: destructive actions only (delete book)
- **Loading states**: skeleton screens on initial list load; spinner on AI identification
- **Notifications**: Sonner toast — bottom-right — use for: save success, delete success, copy to clipboard, AI errors
- **Tables**: paginated (limit 50) · row actions: row click → detail page
- **Navigation**: sidebar with category names, counts, active state = terracotta left border + light terracotta background

## Integrations
- OpenRouter: `google/gemini-2.0-flash` (vision) + `perplexity/sonar-pro` (web enrichment + pricing)
- Resend: invite emails (falls back to console log in dev)

## Tech Stack
Next.js 16 · React 19 · TypeScript · Better Auth · PostgreSQL + Drizzle ORM · shadcn/ui · Tailwind 4 · OpenRouter · Resend
